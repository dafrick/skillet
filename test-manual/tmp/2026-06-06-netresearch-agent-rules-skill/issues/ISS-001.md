# ISS-001: create-skillet crashes on launch — missing font file

**Description**
Running `npx create-skillet` from a cloned skill repo crashes immediately with an ENOENT error. The process cannot find `fonts/ANSI Shadow.flf` inside the installed package directory. The wizard never starts.

**Steps to reproduce**

1. Install Node 24 in a fresh Ubuntu 24.04 container
2. Clone any skill repo (e.g., `git clone https://github.com/netresearch/agent-rules-skill.git`)
3. `cd` into the cloned repo
4. Run `npx create-skillet`
5. Confirm the npx install prompt with `y`

**Expected result**
The interactive wizard launches and presents the first prompt.

**Actual result**
```
Error: ENOENT: no such file or directory, open '/root/.npm/_npx/42e1fff3fd062d19/node_modules/create-skillet/fonts/ANSI Shadow.flf'
    at Module.readFileSync (node:fs:441:20)
    at nodeFiglet.loadFontSync (...create-skillet/dist/run.js:1468:23)
    at me.textSync (...create-skillet/dist/run.js:1188:39)
    at generateWordmark (...create-skillet/dist/run.js:1862:15)
```
Node.js v24.16.0 exits with a non-zero code.

**How encountered**
Step 3 — Run create-skillet.

**Why it is bad / user impact**
The tool is completely unusable on first run via `npx`. A user following the npm README cannot proceed past this point without a workaround. The wordmark rendering is cosmetic but the missing file causes an uncaught exception that kills the process before any wizard prompt appears.

**Severity**
- [x] critical

**Workaround**
The `fonts/` directory was not included in the published npm package. A workaround may be possible by manually creating the directory and supplying the missing font file (e.g., downloading `ANSI Shadow.flf` from the figlet-fonts source), but this is not discoverable from available documentation.
