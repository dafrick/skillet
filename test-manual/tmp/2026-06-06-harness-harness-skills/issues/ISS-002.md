# ISS-002: No multi-skill workflow — wizard packages one skill per run

**Severity:** Hard fail for multi-skill repos; N/A for single-skill repos  
**Discovered:** 2026-06-06, Run 3 (`harness/harness-skills`, T5)  
**Affects:** `create-skillet@0.1.1`

## Description

`create-skillet` has no awareness of multi-skill repositories. The wizard prompts for a single `Skill content path` with a hardcoded default of `skill/`. There is no:

- Skill discovery (scan for all `SKILL.md` files in the repo)
- Skill selection menu
- Batch packaging mode
- Guidance for multi-skill repos in the wizard or output

For `harness/harness-skills`, which contains 53 skills under `skills/*/`, a user who wants to package all skills would need to run `create-skillet` 53 times, each time manually entering a different `skillDir`, creating 53 separate npm packages.

## Steps to Reproduce

1. Clone a multi-skill repo (e.g., `https://github.com/harness/harness-skills`)
2. Run `npx create-skillet`
3. Reach the `Skill content path` prompt
4. Observe: default is `skill/` (wrong); no skill list is offered; no guidance on how to package multiple skills

## Expected Behavior

One of:
- Wizard detects multiple `SKILL.md` files and offers a selection menu (single-skill packaging)
- Wizard offers a "package all skills" mode that creates one package per skill or one combined package
- Wizard output explains the multi-skill limitation and gives guidance

## Actual Behavior

Wizard shows `Skill content path (relative to package root): (skill/)` with no context. User must know the correct path for one specific skill and enter it manually. No path for bulk packaging.

## Workaround

Run `create-skillet` once per skill, entering each skill's path manually. For 53 skills, this is impractical without scripting.

## Impact

T5 (multi-skill) repos are not practically supported by the current wizard UX. Testing was completed by packaging one skill (`skills/ai-operations`) as a representative sample.
